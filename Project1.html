User Login Authentication Implementation Page

Project Purpose

Login security is a crucial aspect of any website to protect the information of users and ensure that unwanted users don't access private parts of websites. Cookies and sessions can be used to store the information of users and ensure that only authenticated users can access specific parts of websites. Initially the Software and Computer Engineering Society (SCE) here on San Jose State campus did not have proper authentication. To fix this problem, I developed a proper user login authentication system which gives different levels of authentication to different levels of users. Depending on the login credentials a specific cookie is assigned to the level of the user. Ex: There are club members, team officers, and executive team members. Each position has different accessibilities and their unique cookie is stored in separate sessions.

Project Description

Technologies Utilized:

Express.js, Nodejs, MongoDB, Bcrypt

Description:

When users create account, level of authentication is selected (Club Member, Team Officer, Executive Staff). Based on selection, users login info stored in specific session stored in MongoDB database. Cookie from session sent to browser which contains session id. When users login in to pages, console checks cookies' session id containing level of authentication for users. For example if a club member tries to login in edit events page which they aren't authenticated for, console sees that their session id doesn't give them authentication for page redirect and doesn't allow them to enter page.
