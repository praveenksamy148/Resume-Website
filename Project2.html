Printing Development Tool Project

Project Purpose

The SJSU Software and Computer Engineering Society (SCE) offers free printing services to its club members. By using our club website and joining the server called Core-V4, students would be able to print any papers they need. However, the overall process took too much time as students would need to login onto the Core-V4 server, have Quasar running, and then print their papers. To solve this issue and overall reduce the time complexity of their process, I decided to create a printing dev tool that utilizes AWS S3 and AWS SQS services. With this printing dev tool, club members will be able to print directly from their terminal running the command npm-run-print and overall save much time on the printing process.

Project Description

Technologies Utilized:

Nodejs, AWS S3, AWS SQS, Quasar

Description:

This project designed a dev tool allowing to directly upload files to SCE printing-dev bucket in AWS S3 and send queue messages to AWS SQS through Nodejs. In addition a method was created allowing for the download and deletion of files from SCE printing dev bucket. By running the dev tool, the file one needs printed gets stored in the S3 bucket and similateneously, a queue message is sent to AWS SQS. Quasar polls SQS for any queue messages every 10 seconds.Each file gets their own separate ID and the ID is read by Quasar to print the specific file.
